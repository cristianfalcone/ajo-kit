# Triage Labels

The engineering skills use five canonical triage roles. This file maps those roles to the actual label strings used in this repo's issue tracker.

| Skill role | Tracker label | Meaning |
| --- | --- | --- |
| `needs-triage` | `needs-triage` | Maintainer needs to evaluate this issue |
| `needs-info` | `needs-info` | Waiting on reporter for more information |
| `ready-for-agent` | `ready-for-agent` | Fully specified, ready for an AFK agent |
| `ready-for-human` | `ready-for-human` | Requires human implementation |
| `wontfix` | `wontfix` | Will not be actioned |

When a skill mentions a triage role, use the corresponding tracker label from this table.

If the GitHub label vocabulary changes, update the right-hand column before running triage automation.
